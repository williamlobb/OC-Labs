### 1. `.env.local` created in repo root

### 2. `isSafeRedirect` — two edge-case bugs patched

- Whitespace-only strings (`'   '`) now return `false` — added `.trim()` check
- Bare hostnames (`'evil.com/path'`) now return `false` — added `startsWith('/')` guard

Final implementation in `src/lib/utils/is-safe-redirect.ts`:

```typescript
export function isSafeRedirect(url: string): boolean {
  if (!url || typeof url !== 'string') return false
  const trimmed = url.trim()
  if (!trimmed) return false
  if (!trimmed.startsWith('/')) return false
  try {
    const parsed = new URL(trimmed, 'http://n')
    return parsed.hostname === 'n'
  } catch {
    return false
  }
}
```

### 3. Import path mismatch fixed

`isSafeRedirect.test.ts` imported from `@/lib/utils/isSafeRedirect` (camelCase).
The actual file is `is-safe-redirect.ts` (kebab-case). Import path corrected.

### 4. `SignupFormInner` test failures — two fixes

- `queryByText(/./` replaced with `queryByRole('paragraph')` — the regex matched
  multiple elements and caused `queryByText` to throw
- Required fields (Name, Email, Password) are now filled before clicking submit in the
  error-state test — jsdom's HTML constraint validation was blocking form submission
  when fields were empty, so the mocked action was never called

### 5. `vi.hoisted()` applied to 5 test files

Files: `login-action.test.ts`, `signup-action.test.ts`, `callback-route.test.ts`,
`GitHubButton.test.tsx`, `middleware.test.ts`

Vitest hoists `vi.mock()` factories before `const` declarations. Plain
`const mockX = vi.fn()` was uninitialized when the factory ran. Fixed by wrapping
all mock declarations in `vi.hoisted(() => ({ mockX: vi.fn() }))`.
